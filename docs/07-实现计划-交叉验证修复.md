## 任务 6：RecipeDetail 食材 owned/missing 标记（P1 — 轻微 UI 增强）

### 现状
`IngredientsSection` 仅显示 `• 食材名 | 数量` 的简单列表，无 owned/missing 区分。

### 设计规格（参考 03-UI设计-v2.md §4.4）

```
═══ 食材清单 ═══
✅ 番茄    2个        ← 绿色 = 已有
✅ 鸡蛋    3个
⬜ 小葱    适量        ← 灰色 = 需购买
```

### 实现方案

需要判断每个食材是否"已拥有"。判断逻辑：
- 菜谱的 `allIngredients` 与用户上次扫描的 `DetectedItem.displayName` 交叉匹配
- 匹配条件：`ingredient.name` 在识别到的食材名称列表中

需要在 `RecipeDetailViewModel` 中计算 owned/missing：
1. 从 `HistoryRepository` 获取最近一次扫描结果
2. 提取扫描到的食材 displayName 列表
3. 对菜谱的每个 ingredient，检查是否在扫描结果中

### 简化方案
为了避免跨层依赖（RecipeDetailViewModel 目前不持有 HistoryRepository），采用轻量级方案：
- 添加 `RecipeDetailViewModel` 的参数 `scannedIngredients: Set<String>`（从 AnalysisViewModel 传入）
- 在 `IngredientsSection` 中接受 `ownedNames: Set<String>` 参数
- UI 层：owned → 绿色 ✅，missing → 灰色 ⬜

如果从导航参数传入扫描食材列表太复杂，降级方案：仅改变 UI 样式（所有食材统一用 `✅` 标记 + 绿色），不做真实的 owned/missing 判断。

**最终采用**：从 `NavGraph` 传入空 Set 作为默认值 → RecipeDetail 接收扫描食材列表参数 → 如果为空则全部显示为 neutral 样式。实际数据流通过 `SavedStateHandle` 或导航参数传递。
