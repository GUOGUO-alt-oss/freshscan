## 0. 模型架构总览（v2 双模型）

```
拍照 → Bitmap
         │
    ┌────┴──────────────────────┐
    │ EfficientDet-Lite0         │  检测阶段
    │ (COCO 预训练, 320×320)     │  → bounding boxes
    └────┬──────────────────────┘
         │ 对每个检测框 crop
    ┌────┴──────────────────────┐
    │ MobileNetV3 (260 类)       │  识别阶段 (Fruits-360 训练)
    │ 输入: [1,224,224,3]       │  → "红富士苹果" / "樱桃番茄"
    │ 输出: [1,260] logits      │
    └────┬──────────────────────┘
         │
    ┌────┴──────────────────────┐
    │ MobileNetV3 (18 类 v1)    │  新鲜度阶段 (辅助)
    │ 输入: [1,224,224,3]       │  → FRESH / ROTTEN
    │ 输出: [1,18] logits       │
    └────┬──────────────────────┘
         │
    DetectedItem { label, freshness, bbox, isCookable }
```

**模型规格:**

| 模型 | 用途 | 类别 | 体积 | 延迟 |
|------|------|------|------|------|
| EfficientDet-Lite0 | 检测框 | COCO 80 类 | 5.2MB | ~80ms |
| MobileNetV3-260 | 品类识别 | Fruits-360 260 类 | ~5.0MB | ~50ms |
| MobileNetV3-18 (v1) | 新鲜度 | 18 类新鲜/腐烂 | 3.6MB | ~50ms |
| **总计** | | | **~14MB** | **~180ms** |

### 3.3 菜谱引擎

```kotlin
// data/recipe/RecipeEngine.kt
// data/recipe/LabelNormalizer.kt — 标签归一化层（v2.4 新增）

class LabelNormalizer @Inject constructor() {
    // 从 labels_v2.txt 的 displayName → 菜谱食材名 的映射
    // 例: "樱桃番茄" → ["番茄"], "红富士苹果" → ["苹果"]
    private val mapping: Map<String, List<String>> by lazy {
        loadMapping()  // 可由 labels_v2.txt 扩展字段或独立 JSON 维护
    }

    fun normalize(fruitLabel: String): List<String> {
        return mapping[fruitLabel] ?: listOf(fruitLabel)
    }
}

class RecipeEngine @Inject constructor(
    @ApplicationContext private val context: Context,
    private val labelNormalizer: LabelNormalizer  // v2.4 新增
) {
    private val presetRecipes: List<Recipe> by lazy {
        loadPresetRecipes(context)  // 从 assets/recipes/preset_recipes.json 加载
    }

    fun recommend(
        items: List<DetectedItem>,
        profile: TasteProfile? = null
    ): RecipeResult {
        val cookable = items.filter { it.isCookable }

        if (cookable.isEmpty()) {
            return RecipeResult(
                recipes = emptyList(),
                note = "检测到的全是水果，建议直接食用 🍎"
            )
        }

        // 标签归一化: Fruits-360 英文标签 → 菜谱中文食材名
        // 例: "Tomato_Cherry_Red" → displayName "樱桃番茄" → 标准化为 "番茄"
        val ingredientNames = cookable.map { item ->
            labelNormalizer.normalize(item.label)  // 返回 ["番茄", "黄瓜", "西兰花"]
        }.flatten().distinct()

        // 按食材匹配数量排序
        val matched = presetRecipes
            .map { recipe ->
                recipe to recipe.matchIngredients.count { it in ingredientNames }
            }
            .filter { (_, score) -> score > 0 }
            .sortedByDescending { (_, score) -> score }
            .map { (recipe, _) -> recipe }

        return RecipeResult(recipes = matched, note = null)
    }
}

data class RecipeResult(
    val recipes: List<Recipe>,
    val note: String?
)
```

## 4. 推理管线（v2.4 重写 — 修复坐标空间 + 三阶段管线）

**管线架构（与 §0 一致）：**

```
用户拍照完成 → Bitmap (max 1920px, 以 inSampleSize=2 加载以防 OOM)

AnalysisViewModel.infer(bitmap) {
    launch(Dispatchers.Default) {

        // ═══ 阶段 1: 多目标检测 ═══
        val detectInput = ImagePreprocessor.resize(bitmap, 320, 320)
        val (boxes, classes, scores) = efficientDet.detect(detectInput)

        // NMS 后处理: 去重 + 低置信度过滤
        // score 阈值降至 0.4（考虑到检测模型未在果蔬数据上微调）
        val validBoxes = DetectionPostprocessor.nms(
            boxes, classes, scores,
            iouThreshold = 0.5,
            scoreThreshold = 0.4
        )

        // ═══ 阶段 2: 对每个检测框做品类识别 + 新鲜度分类 ═══
        val items = validBoxes.map { box ->
            // 关键: 坐标反归一化 — box 在 320×320 空间，需映射回原始 bitmap 空间
            val scaleX = bitmap.width.toFloat() / 320f
            val scaleY = bitmap.height.toFloat() / 320f
            val cropLeft  = (box.left * scaleX).toInt().coerceIn(0, bitmap.width)
            val cropTop   = (box.top  * scaleY).toInt().coerceIn(0, bitmap.height)
            val cropRight = (box.right  * scaleX).toInt().coerceIn(0, bitmap.width)
            val cropBottom = (box.bottom * scaleY).toInt().coerceIn(0, bitmap.height)

            // 跳过无效框（宽/高 < 20px）
            if (cropRight - cropLeft < 20 || cropBottom - cropTop < 20) return@map null

            val crop = Bitmap.createBitmap(bitmap, cropLeft, cropTop,
                                           cropRight - cropLeft, cropBottom - cropTop)

            // 品类识别: MobileNetV3-260 (Fruits-360 训练)
            val input260 = ImagePreprocessor.bitmapToTensorBuffer(crop, 224, 224)
            val logits260 = classifier260.classify(input260.buffer)
            val (label, displayName, confidence, isCookable) =
                modelMapper260.mapToLabelInfo(logits260!!)  // 查 labels_v2.txt

            // 新鲜度分类: MobileNetV3-18 (v1 模型复用)
            val input18 = ImagePreprocessor.bitmapToTensorBuffer(crop, 224, 224)
            val logits18 = classifierFreshness.classify(input18.buffer)
            val freshness = modelMapper18.mapToFreshness(logits18!!)

            DetectedItem(
                id = UUID.randomUUID().toString(),
                label = label,                     // "Tomato_Cherry_Red"
                displayName = displayName,         // "樱桃番茄"
                freshnessLevel = freshness,         // FRESH / ROTTEN / UNCERTAIN
                confidence = confidence,            // [0.0, 1.0]
                bbox = RectF(                       // 归一化坐标 [0, 1]
                    box.left / 320f, box.top / 320f,
                    box.right / 320f, box.bottom / 320f
                ),
                isCookable = isCookable             // 来自 labels_v2.txt
            )
        }.filterNotNull()  // 移除无效框

        _uiState.update { it.copy(items = items, isAnalyzing = false) }
    }
}
```

**性能预估（串行，单框 ~130ms；含 NMS ~180ms 总推理）：**

| 阶段 | 操作 | 耗时 |
|------|------|------|
| 检测 | EfficientDet-Lite0 推理 (320×320, CPU 4线程) | ~80ms |
| 后处理 | NMS + 坐标反归一化 | ~5ms |
| 分类 (×N) | MobileNetV3-260 推理 (224×224) + MobileNetV3-18 (224×224) | ~50ms + ~50ms = ~100ms/框 |
| 2框场景 | 总计 | ~80 + 5 + 200 = ~285ms |
| 5框场景 | 总计 | ~80 + 5 + 500 = ~585ms (建议并行化) |

> **优化建议：** 若检测框 > 3 个，将每个框的两个分类任务并行执行（`coroutineScope { async { ... } }`），可将 N 框分类时间压至 ~100ms（与框数无关）。

**2 秒粒子动画完全覆盖所有推理场景。**

## 7. 错误处理矩阵

| 场景 | 表现 | 处理 |
|------|------|------|
| 系统相机无响应 | — | 30 秒超时 → 回到首页 + Snackbar "相机未响应" |
| 拍照失败 | — | onActivityResult(RESULT_CANCELED) → 回到首页 |
| 模型加载失败 | 分析页白屏 | 检测到 `isModelReady == false` → Snackbar "AI 引擎启动失败" + 重试按钮 |
| 检测无结果 (0 框) | 粒子动画结束无食材 | BottomSheet 显示 "未检测到果蔬，请重新拍照" + [重拍] 按钮 |
| 检测结果全水果 | 有框但 isCookable 全 false | BottomSheet 显示食材 + 提示 "建议直接食用" + 无[找菜谱]按钮 |
| 菜谱匹配为 0 | [找菜谱] 后空列表 | LazyColumn 显示 "未找到匹配菜谱，试试其他分类筛选" |
| 拍照权限拒绝 | — | Snackbar "请在设置中开启相机权限" + 跳转系统设置按钮 |
| OOM (大照片) | 崩溃 | Bitmap 采样 `inSampleSize=2`，最大尺寸限制 1920px |
| Process Death | 系统相机期间 Activity 被杀死 | `SavedStateHandle` 保存 `photoUri`，恢复后直接进入分析页，跳过重新拍照 |
| MediaPipe/TFLite 冲突 | 编译或运行时 crash | 优先使用 MediaPipe 内置 TFLite runtime；如冲突则降级为独立 TFLite + 自实现 NMS |

## 8. 模型加载策略（v2.4 新增）

### 8.1 加载时机

| 策略 | 优点 | 缺点 | 采用 |
|------|------|------|------|
| Application.onCreate 全量加载 | 拍照时无需等待 | 冷启动 +500-800ms，低端机可能 ANR | ❌ |
| 分析页进入时懒加载 | 不影响冷启动 | 首次拍照等待 1-2s（2s 粒子动画可覆盖）| ✅ **推荐** |
| 后台预加载（首页空闲时） | 兼顾两者 | 实现复杂度高 | 备选 |

**采用懒加载策略：** 用户点击「扫描食材」进入分析页时开始加载模型。2 秒粒子动画足以覆盖模型加载 + 推理时间。

### 8.2 GPU Delegate 分配

三个模型总磁盘 14MB，GPU delegate 下内存更高：

| 设备 RAM | EfficientDet-Lite0 | MobileNetV3-260 | MobileNetV3-18 |
|----------|-------------------|-----------------|-----------------|
| ≥6GB | GPU delegate | GPU delegate | GPU delegate |
| 4-6GB | GPU delegate | CPU (XNNPACK 4线程) | CPU (XNNPACK 4线程) |
| <4GB | CPU (XNNPACK 4线程) | CPU (XNNPACK 4线程) | CPU (XNNPACK 4线程) |

```kotlin
// data/inference/ModelLoader.kt (新增)
class ModelLoader @Inject constructor(
    @ApplicationContext private val context: Context
) {
    private val isLowRamDevice by lazy {
        val activityManager = context.getSystemService(Context.ACTIVITY_SERVICE) as ActivityManager
        activityManager.memoryClass < 256  // < 4GB
    }

    fun createInterpreter(modelPath: String, useGpu: Boolean = !isLowRamDevice): Interpreter {
        val options = Interpreter.Options().apply {
            if (useGpu) {
                try {
                    addDelegate(GpuDelegate())
                } catch (e: Exception) {
                    Logger.w("ModelLoader", "GPU delegate failed, falling back to CPU", e)
                    setNumThreads(4)
                    setUseXNNPACK(true)
                }
            } else {
                setNumThreads(4)
                setUseXNNPACK(true)
            }
        }
        return Interpreter(loadModelFile(modelPath), options)
    }
}
```

### 8.3 卸载策略

- 退出分析页（onCleared）→ 延迟 30 秒后关闭 `Interpreter`（避免频繁进出重新加载）
- 内存警告 `onTrimMemory(TRIM_MEMORY_RUNNING_LOW)` → 立即关闭所有 Interpreter
- 首页不持有模型引用
