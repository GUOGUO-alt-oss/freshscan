## 4. 架构与设计评估

### 4.1 架构合规性

| 维度 | 评分 | 说明 |
|------|------|------|
| 分层隔离 | ⭐⭐⭐⭐⭐ | 领域层零 Android 依赖；数据层接口在领域层定义；表现层仅依赖 ViewModel |
| 依赖注入 | ⭐⭐⭐⭐⭐ | ModelConfig → ModelMapper/ImagePreprocessor/TFLiteClassifier 全部通过构造函数注入，无自行实例化 |
| 单向数据流 | ⭐⭐⭐⭐⭐ | StateFlow 驱动 Compose，`MainUiState` 不可变 data class，`update {}` 原子修改 |
| 存储库模式 | ⭐⭐⭐⭐⭐ | Camera/Inference/History 三层 Repository 接口完整定义，实现层正确分离 |

### 4.2 设计亮点

1. **处理器适配的 NV21 转换**（`ImagePreprocessor.kt`）：正确区分半平面（pixelStride=2）和平面（pixelStride=1）布局，覆盖 Qualcomm、MediaTek、Exynos、Tensor 全系 SoC。buffer position 保存/恢复机制避免下游读取偏移。

2. **推理管道降级链**：GPU delegate → 4 线程 CPU + XNNPACK → 低功耗模式（500ms 间隔），完整的性能自适应链路。

3. **Raw-logit 预检**（`ModelMapper.kt`）：在 Softmax 前检查 max logit 幅度，低于 `MIN_LOGIT_FOR_CONFIDENCE`（4.0）直接标记 UNKNOWN。避免了非果蔬画面（手掌、墙壁、屏幕）通过 Softmax 归一化获得虚假高置信度的问题。

4. **稳定性缓冲**（`MainViewModel.kt`）：连续 3 帧类别+新鲜度一致才标记稳定，有效过滤单帧抖动。Mutex 保护保证原子性。

5. **Guide 引导持久化**：SharedPreferences 存储 `guide_shown` 标记，首次启动显示后永久跳过，不再每次启动重复出现。

### 4.3 性能评估

| 指标 | 值 | 评价 |
|------|-----|------|
| 推理 FPS | ~3（300ms 间隔） | ✅ 平衡了实时性与功耗 |
| 单帧推理超时 | 800ms | ✅ 合理，防止阻塞管线 |
| 模型体积 | FP32 ~3.60MB | ✅ 内嵌模型，APK 增量可控 |
| 历史记录上限 | 50 条 | ✅ 配合自动 trim 防止数据库膨胀 |
| 缩略图尺寸 | 256px | ✅ 平衡清晰度与存储 |
| 线程安全 | synchronized(interpreterLock) | ✅ TFLite 所有 interpreter 操作受保护 |
