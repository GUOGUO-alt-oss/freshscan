# 「鲜识 v2.0」产品需求文档（PRD）

> 版本：v2.4 | 作者：陈润扬 | 日期：2026-06-17 | 审查：Claude Code（详见 05-文档审查报告-v2.md）
> 基线：v1.0 已交付（18 类实时分类 + 历史记录，精度 98.64%）
> v2 核心升级：品类识别从 9 种 → **260 种**（Fruits-360 数据集）；新增多框检测 + 100+ 菜谱智能推荐；UI 升级至 Material Design 3

---

## 1. 产品定位升级

| 维度 | v1.0（已交付） | v2.0（本次升级） |
|------|-------------|-------------|
| 一句话 | "打开相机照一照，果蔬好坏即刻知道" | "扫一扫冰箱，告诉你怎么吃最健康" |
| 核心交互 | 实时相机 + 逐帧推理 | 系统相机拍照 → AI 分析 → 结果弹出 |
| 核心模型 | 单目标分类（MobileNetV3） | 多目标检测（EfficientDet）+ 新鲜度分类（MobileNetV3，复用） |
| 用户心智 | 「判断工具」| 「厨房 AI 助手」— 从判断到决策的完整闭环 |
| v1 兼容 | — | v1 功能保留可切换，历史/详情页复用 |

## 2. 核心交互流程

```
🏠 首页 → [扫描食材] → 📷 系统相机 → ✨ 分析页

详细路径:

首页（功能全展示）
  ├── 扫描食材 Hero 卡片（点击 → 系统相机）
  ├── 最近扫描摘要（基于上次结果）
  ├── 筛选 Chip 行（减脂 / 家常 / 快手 / 汤羹 / 凉拌）
  └── 菜谱卡片流（LazyColumn，基于所有已扫描食材）

      ↓ 点击「扫描食材」

系统相机 (Intent: ACTION_IMAGE_CAPTURE)
      ↓ 拍照完成 → Bitmap

分析页
  Phase 1: 照片全屏 + 粒子光效动画（2 秒）→ "AI 正在分析..."
  Phase 2: 动画结束 → 暗色遮罩减淡 → BottomSheet 滑入（COLLAPSED 态，~220dp）
           → 食材列表（名称 + 新鲜度 + 缩略图）
           → [找菜谱] 按钮
  Phase 3: 点击「找菜谱」/ 上滑拖拽条 → BottomSheet 展开（HALF 态，~450dp）
           → 食材列表 + 筛选 Chip 行
  Phase 4: 继续上滑到底 → BottomSheet 全屏（FULL 态）
           → 照片缩至 120dp 顶栏
           → 食材列表 + 筛选 Chip + 菜谱卡片流（与首页相同布局）
           → 下滑回退 → COLLAPSED → 再下滑 → 关闭

菜谱详情页（独立路由 recipe/{id}）
  ├── Hero 大图
  ├── 食材清单（已有 ✓ / 需购买 ⬜）
  ├── 烹饪步骤（编号 + 每步可选计时）
  ├── 营养摘要（彩色进度条）
  └── 操作：[收藏] [缺的加入购物清单]
```

## 3. 功能需求池

### P0 — 核心新增

| 功能 | 说明 | 验收标准 |
|------|------|---------|
| 系统相机拍照 | `ACTION_IMAGE_CAPTURE` 调用原生相机 | 拍照成功 → 返回 1920px Bitmap |
| 多目标检测 | EfficientDet-Lite0 单次检测画面中所有果蔬 | 检出 ≥ 1 个果蔬，box 置信度 > 0.5 |
| 新鲜度分类 | 每个检测框裁剪 → MobileNetV3 推理 | 正确输出 18 类新鲜/腐烂 + 置信度 |
| 粒子光效动画 | 拍照后 200 粒子从边缘向中心汇聚（2 秒） | 60fps 不掉帧，动画覆盖推理等待 |
| 底部结果弹出 | BottomSheet 三态（半屏食材 → 全屏菜谱 → 收起） | 滑动手势流畅，与 LazyColumn 无冲突 |
| 菜谱匹配引擎 | 预置 100+ 道菜谱 JSON + 食材关键词匹配 | 1+ 食材命中即返回结果，按匹配数降序 |
| 分类筛选 | FilterChip：减脂/家常/快手/汤羹/凉拌 | 点击筛选 LazyColumn 即时刷新 |
| M3 UI | Material Design 3，动态取色，28dp 大圆角 | 首页 / 分析 / 详情 / 口味 / 清单 / 历史 / 设置 共 8 页 |

### P1 — 体验增强

| 功能 | 说明 |
|------|------|
| 口味档案 | 辣度滑块(0-5)、忌口多选、烹饪方式偏好、每餐热量目标 |
| 食材购物清单 | 菜谱中"需购买"食材一键加入，列表页勾选管理 |
| 菜谱详情 | 步骤卡片 + 每步独立计时器 BottomSheet |
| 营养摘要 | 热量 + 蛋白质/碳水/脂肪/纤维 彩色进度条 |
| 菜谱收藏 | 用户标记喜欢的菜谱，首页优先展示 |

### P2 — 远期

| 功能 | 说明 |
|------|------|
| 端侧 LLM 菜谱 | Gemma-3-1B 生成个性化菜谱（框架预留，当前降级为预置菜谱） |
| 保质期预测 | 置信度变化曲线 → 估算剩余可食用天数 |
| 食物浪费追踪 | 记录丢弃食物 + 碳排放计算 |

## 4. 新增用户画像

| 用户 | 年龄 | 痛点 | 场景 |
|------|------|------|------|
| 减脂人群 | 18-35 | 冰箱有食材但不知道低热量搭配 | 拍冰箱 → 筛选减脂 → 选菜谱 → 计算热量 |
| 烹饪小白 | 18-28 | 看到食材不会搭配、不知烹饪时间 | 拍食材 → 选 ≤5 步菜谱 → 跟计时器做菜 |
| 日常做饭者 | 25-45 | 懒得想今天吃什么、缺食材不知道 | 拍冰箱 → 浏览推荐 → 加购物清单 |

## 5. 成功指标

| 指标 | 目标 |
|------|------|
| 多目标检出率（≥1 个果蔬） | ≥ 95%（正常光照桌面场景） |
| 菜谱首次匹配成功率 | ≥ 90%（至少返回 1 道菜谱） |
| 拍照→出结果 耗时 | < 4 秒（含 2 秒动画） |
| 扫描→选菜→加清单 操作步骤 | ≤ 5 步 |
| v1 功能兼容 | 历史/详情页无回归，Settings 可切换回 v1 |

## 6. v1 兼容策略

| 组件 | 策略 |
|------|------|
| 首页 | v2 新首页为默认，v1 `MainScreen` 保留标记 `@Deprecated` |
| 历史记录 | `history` 路由 + `HistoryScreen` + `HistoryViewModel` 全部复用 |
| 识别详情 | `detail/{id}` 复用，增加 `DetectedItem` → `RecognitionResult` 转换 |
| 模型 | MobileNetV3（新鲜度分类）保留；新增 EfficientDet（检测） |
| 设置 | `Settings` 页增加「使用经典模式」开关，切换回 v1 首页 |
